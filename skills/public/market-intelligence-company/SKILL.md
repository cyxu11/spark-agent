# 中海油气电集团公司经营数据分析

> 用户画像:战略规划部门(看 F 基本面)/ 销售部门(看 T+F)/ 央企合规审计(看 R 政策面)/ 财务部门(看 F 营收毛利)

## 一、何时使用本 skill

✅ 触发场景:
- 中海油 LNG 进口量(月度+年度,分气源国:澳大利亚/卡塔尔/马来西亚 等)
- 长协 vs 现货 进口比例
- 气电集团销售量(气源/气液态/区域 三维)
- 接收站罐存与库容利用率(日度)
- 月度营收毛利(分业务类型)
- 季度气源市场份额(LNG/海气/煤层气/外采气/文-23)
- 销售大区管道气价(月度,9 大区)
- 分区域资源池价格(华南/华东/华北/福建)

❌ 不适用 → 改用其他 skill:
- 国际天然气价格(JKM/TTF/HH) → `market-intelligence-international`
- 国内 LNG 挂牌价(14 区域) / 全国天然气消费 → `market-intelligence-domestic`
- 价格换算 → `market-intelligence-converter`

⚠️ **跨域优先级**:用户问"中海油 LNG 进口跟 JKM 价格的关系" → 优先命中 **company**(公司经营优先)。

## 三、🚨 特别警示:经营数据敏感性(审计红线)

经营数据高度敏感,**任何编造都是审计风险**:
- **禁编造进口量 / 销售量 / 长协占比 / 毛利率 / EVA / 市场份额**
- `executeSqlToolCallback` 返回 0 行 / 缺数据时,**改为方向性描述**,不得自行填数字
- 严格遵守 Step 5 的 S0 数据原样回显约束
- 数量 (`进口量`) 与占比 (`market_share`) 是不同维度,**清洗 query 时务必区分清楚**(详见 Step 1)

## 四、工作流(严格顺序)

> **数据链路总览**:Step 1(query 清洗) → Step 2(`listAgentsToolCallback` 发现 agentId) → Step 3(`nl2SqlToolCallback` 生成 SQL) → Step 4(`executeSqlToolCallback` 执行 SQL 拿数据) → Step 5(数据明细段) → Step 6(MRTF 分析,若需) → Step 7(自检)。**Step 2-4 三个工具各调用 1 次,串行不并行**。

### Step 1:用户 query 预处理(LLM 内化,不输出 JSON)

把原始用户问句**就地清洗成一句完整、可直接作为 NL2SQL 自然语言入参 (`naturalQuery`) 的中文 query 字符串**。

清洗规则(详细见 `references/tool-routing.md`):

1. **🔴 关键区分:`数量` vs `占比`**(经营数据特有,清洗时必须辨别)

   | 用户提到的词 | 应识别为 | 清洗示例 |
   |------------|---------|---------|
   | `进口量` / `销售量` / `万吨` / `亿立方米` | 数量(绝对量) | "2024 进口量" → 清洗为"2024 年中海油 LNG 进口量(万吨)" |
   | `占比` / `比例` / `结构` / `份额` / `%` | 占比(百分比) | "气源结构" → 清洗为"2024 年中海油气源结构占比" |
   | 同时问数量 + 占比 | 拆成两个清洗后的 query?不,合并表达 | "进口量及长协占比" → 保留双指标表达 |

2. **🔴 时间粒度识别**:
   - `市场份额` 是**季度**粒度 → 清洗时保留"季度"或具体"Q1/Q2"
   - `进口量` / `销售量` / `营收` 是**月度+年度**粒度
   - `罐存` 是**日度**粒度
   - 用户说"近 3 月" / "Q1" 等 → 保留原表达,**不擅自换粒度**

3. **气源国规范化**:`澳大利亚` / `卡塔尔` / `马来西亚` / `印尼` / `尼日利亚` / `俄罗斯` / `秘鲁` / `阿曼` / `塞内加尔` / `其他`——用户用别名(如"澳洲")→ 翻成"澳大利亚"

4. **进口类型识别**:`长协` / `现货`(混用别名"长期协议"/"即期"等 → 翻成"长协"/"现货")

5. **业务类型 / 气源 / 气液态**:销售场景的关键维度,保留原词

6. **资源池区域**:`华南` / `华东` / `华北` / `福建`(资源池价场景)

7. **时间范围补全**:
   - "近 3 月" → "过去三个月"
   - "今年以来" → "2026 年 1 月至今"
   - "去年" → "2025 年全年"
   - 未提到时间 → 不补

8. **保留原意,禁加约束**:不擅自加 "≤ 5 条" / "仅长协不含现货" 等用户未提及的过滤

示例:
- `"中海油 2024 年 LNG 进口"` → `"2024 年中海油 LNG 进口量(分气源国,长协与现货)"`
- `"气源结构占比"` → `"2024 年中海油气源结构市场份额占比(季度)"`
- `"澳大利亚 LNG 占比"` → `"2024 年澳大利亚气源 LNG 进口占比"`
- `"华南资源池价"` → `"2024 年华南区域资源池价格(季度)"`
- `"近 3 月毛利"` → `"过去三个月中海油气电集团月度营收与毛利"`

### Step 2:发现 NL2SQL 智能体 ID(`listAgentsToolCallback`)

调用 `listAgentsToolCallback` 1 次,挑出最契合 query 主题的智能体,抽取 `id` 作为 `agentId`。

**调用参数**:
```json
{
  "agentListRequest": {
    "keyword": "中海油",
    "status": "published"
  }
}
```

**`keyword` 选择说明**:本 skill 固定用 `"中海油"`,匹配未来 MCP 网关里以"中海油"命名/描述/标签的智能体(如"中海油经营智能体"、"中海油气电分析智能体"、"中海油 LNG 进口智能体"等)。**如需改 keyword**:同时改本文 + `references/tool-routing.md § Step 2`。

**匹配规则**:
- 返回数组非空 → 取 `description` / `category` / `tags` 与 query 域最贴合的一项(经营 / 进口 / 销售等);并列时取最新 `updateTime`
- 返回 `[]` → 回退 `keyword="经营"` 再搜;仍为空则 `keyword=""`;最终为空中止,输出"暂无可用智能体,数据查询不可用"
- **🚨 严禁**:LLM 凭空编造 `agentId`,必须从本次 `listAgents` 实际返回中读取

抽出 `id` 后,**所在 turn 内复用**。

### Step 3:生成 SQL(`nl2SqlToolCallback`)

```json
{
  "nl2SqlRequest": {
    "agentId": "<Step 2 抽取的 id,字符串形态>",
    "naturalQuery": "<Step 1 清洗后的 query>"
  }
}
```

**🚨 硬约束**:
- 返回的 SQL **仅作为 Step 4 的输入**,**严禁向用户回显**(经营数据 SQL 包含敏感表结构,审计风险)
- 若返回 SQL 异常(空字符串 / 缺 WHERE / 不存在表名) → 中止 Step 4,输出"SQL 生成异常,请缩窄 query 范围或换种表述"
- 不要修改 SQL 字符串

### Step 4:执行 SQL(`executeSqlToolCallback`)

```json
{
  "request": {
    "agentId": "<同 Step 3>",
    "sql": "<Step 3 返回的 SQL 字符串>"
  }
}
```

**返回处理**:
- 行数 ≥ 3 → 进入 Step 5 用表格呈现
- 行数 ≤ 2 → 进入 Step 5 用自然语言句呈现
- 行数 = 0 → "未检索到对应数据,请调整时间 / 气源国 / 业务类型范围",**禁编造行**填充(经营审计红线)
- 工具报错 → "数据执行异常,可稍后重试"

### Step 5:数据明细段输出(retriever 风格)

**🚨 流程硬门控(Output-Gate,经营审计场景必守)**
- Step 5 **必须作为完整独立段先于 Step 6 任何 MRTF 章节输出**(数据表格 + 末尾的"据本平台经营数据,共 N 条"统计行)
- **严禁**省略 Step 5 直接进入"M-宏观面/R-政策面..."等 MRTF 章节
- **严禁**把 Step 5 数据表格合并 / 移动 / 改写到 Step 6 T-Trend 表格里 — T-Trend 是经营数据的**二次引用**用于分析语境,不能替代 Step 5 的明细审计层
- 若用户只要数据明细(不要研判),Step 5 输出完即停止;若用户要研判,Step 5 + Step 6 都要有,顺序不可调换
- 工程检测点:回答首屏必须先看到经营数据表格,再看到"### 一、宏观面"等 H3 标题;否则视为 S0 违规重写

**S0 经营数据原样回显零容忍**(审计风险等级最高)
- 你**只能**输出 Step 4 `executeSqlToolCallback` 实际返回的具体经营数值
- **强制原样回显**:Markdown 表格每行严格复制工具返回的字段值
- **禁编造时间窗口**:若 SQL 返回的是 `2024 年 1-3 月` 明细,**不准编造** `2024 年 Q4` / `2025 年` 等不在返回里的行
- **禁编造经营指标**:进口量 / 销量 / 长协占比 / 现货溢价 / 毛利率 / EVA / 市场份额等所有数值都必须来自工具返回

**禁自行聚合**:SQL 已经决定是否聚合(SUM / AVG / GROUP BY 由 NL2SQL 智能体生成,经营场景下尤为关键)。**严禁 LLM 再对返回行做算术汇总**(如把真实合计 192 / 119 / 142 错算成 285 / 268 / 276)。如需聚合视图但 SQL 未给,直接呈现原始行,不要补算。

**输出格式**:
- ≥ 3 行结构化数据:GFM markdown 表格(表头中文 + `|---|---|` 分隔行)
- ≤ 2 行单点数据:自然语言段落
- 末尾追加「据本平台经营数据,共 N 条记录,呈现最近 K 条」
- 数值保留工具返回原始精度,**禁四舍五入**
- 数据点 > 5 时仅呈现最近 5 条

**禁止**:
- 暴露 SQL 字符串 / 表名 / `listAgents*` / `nl2Sql*` / `executeSql*` 工具名 / `mcp_*` / `agentId` 等内部标识(审计场景下尤其敏感)
- 输出累计涨跌幅 / 趋势结论(那是 Step 6 分析段的工作)
- HTML `<table>` / 伪表格 / `#` 标题 / ` ``` ` 代码块

### Step 6:MRTF 分析段(analyst 风格)

**触发条件**:用户问题需要研判(如「走势如何」「趋势分析」「为什么」「研判」「分析」等)。
**跳过条件**:用户只要数据明细(如「给我 2024 进口量」「列一下罐存」),不写 MRTF。

#### Step 6.1:联网搜索

调用本平台内置 `web_search` 工具,**仅 1 个核心关键词**(总结 query 主体后直接搜 1 次),获取国资委 / 央企监管 / 行业资讯。

#### Step 6.2:MRTF 5 块(经营域)

```
### 一、宏观面 (M-Macro)
(100-250 字。国资委央企监管 / 行业整体景气 / 进口端汇率与油价 — 主要联网)

### 二、政策面 (R-Regulation)
(100-250 字。国资委 / 央企合规 / 行业政策 / 双碳 / 长协占比目标 — 主要联网)

### 三、技术面 (T-Trend)
(100-250 字。集团长协 vs 现货价格走势 / 罐容利用率 / 资源池价格区域差 — 本平台数据)

[T 章节第一表格:**对 Step 5 数据明细的二次引用**(N ≤ 10 行),用于支撑技术面分析语境;**不替代 Step 5 独立数据段** — Step 5 必须在本章之前已经独立输出]

### 四、基本面 (F-Fundamental)
(100-250 字。集团 LNG 进口量 / 销售量 / 客户结构 / 市场份额 / 营收 / 毛利率 — 本平台数据)

| 维度 | 数值 | 同比变化 | 备注 |
| 至少 3 行 |

**综合研判**:(150-300 字。方向性结论 + 风险机遇 + 操作建议(规划/销售/合规审计/财务))
```

#### Step 6.3:强制约束(含经营敏感约束)

- **S0 经营数据敏感**:T/F 表格严格复制 Step 5,**严禁臆造毛利率 / EVA / 销量 / 长协占比等具体经营指标**;数据缺失 → 改为方向性描述
- **S1**:5 H3 中文数字编号(`一/二/三/四`)
- **S2**:综合研判 150-300 字 ±20%,三要素齐全
- **S3**:T/F 章节至少 1 个 markdown 表格,数据行 ≥ 3(双边对比 ≥ 2)
- **S4**:联网关闭场景(candidate_pool 为空)M/R 段标注「本段联网资讯未启用」且 ≥ 50 字,T+F 合计 ≥ 400 字
- **`**综合研判**` 前一字符必为 `\n` 或文档第 0 字符**
- **`[数据]` 标识硬约束**:每次引用本平台经营数据具体数值必须在末尾追加 `[数据]`
- **机构引用真实性**:仅当 `web_search` 实际返回某机构内容时用 `据 [国资委](URL)`;无真实返回 → 「据本平台经营数据」

### Step 7:输出前自检

按 `shared/references/output-self-check.md` 逐项核对。

## 七、硬约束摘要(经营审计风险,贴墙)

1. **数据链路 3 工具串行调用,各 1 次**:`listAgentsToolCallback` → `nl2SqlToolCallback` → `executeSqlToolCallback`;MRTF 阶段额外 1 次 `web_search`

5. **数量 vs 占比 严格区分**(清洗 query 时必须辨别"万吨" / "占比"语义)

8. **行数 = 0 不补编造**,改为方向性描述
9. **分析段 S1**:5 H3 中文数字编号
10. **分析段 S2**:综合研判 150-300 字,三要素齐全
11. **分析段 S3**:T/F 表格 ≥ 3 行
12. **分析段 S4**:联网关闭场景 M/R ≥ 50 字 + T/F 合计 ≥ 400 字
13. **`**综合研判**` 前必为 `\n`**
14. **本平台数据末尾 `[数据]` 标识**
15. **禁:SQL / 表名 / `listAgents*` / `nl2Sql*` / `executeSql*` / `agentId` / `mcp_*` / `team3` / `sub-3-*` / `cnooc_*` / `/mnt/skills/...` 路径 / `read_file(...)` 调用语法 等内部组件名**
