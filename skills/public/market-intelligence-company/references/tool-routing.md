# 经营域 NL2SQL 链路调用指南

> 经营数据查询底层走 **3 工具串行链路**:agent 发现 → NL2SQL 生成 → SQL 执行。**经营数据敏感性最高**,本文重点放在 query 清洗阶段如何精准辨别"数量 vs 占比"、"时间粒度"、"长协 vs 现货"等关键经营维度。

## 一、工具链概览

| 顺序 | 工具名 | 入参 | 返回 | 单次调用 |
|------|-------|-----|------|---------|
| Step 2 | `listAgentsToolCallback` | `{agentListRequest: {keyword, status}}` | 智能体数组(含 `id` / `name` / `description` / `category` / `tags`) | 每问题 1 次 |
| Step 3 | `nl2SqlToolCallback` | `{nl2SqlRequest: {agentId, naturalQuery}}` | SQL 字符串 | 每问题 1 次 |
| Step 4 | `executeSqlToolCallback` | `{executeSqlRequest: {agentId, sql}}` | 行数据(JSON 数组) | 每问题 1 次 |

辅助工具:`web_search`(本平台内置联网搜索)仅在 Step 6.1 MRTF 分析阶段调用 1 次,用于 M / R 维度。

## 二、Step 1 query 清洗规则(经营域专属)

### 2.1 🔴 数量 vs 占比(经营审计区分)

| 用户提到的词 | 应识别为 | 清洗示例 |
|------------|---------|---------|
| `进口量` / `销售量` / `万吨` / `亿立方米` / `罐存量` | **数量(绝对量)** | "2024 进口量" → "2024 年中海油 LNG 进口量(万吨,分气源国)" |
| `占比` / `比例` / `结构` / `份额` / `%` | **占比(百分比)** | "气源结构" → "2024 年中海油气源结构市场份额占比(季度)" |
| `营收` / `毛利` / `毛利率` / `EVA` | **财务指标** | "近 3 月毛利" → "过去三个月中海油气电集团月度营收与毛利" |

❌ 严禁犯的错:
- 用户问"某月进口量",清洗成"某月气源占比"(数量被混用成占比)
- 用户问"市场份额",清洗成"销售量"(占比被混用成数量)

### 2.2 🔴 时间粒度规范化

| 业务对象 | 粒度 | query 清洗需保留 |
|---------|------|---------------|
| `市场份额` | **季度** | "季度" / "Q1" / "Q2" 等季度表达 |
| `进口量` / `销售量` / `营收` / `毛利` | **月度+年度** | "月度" / 年份 + 月份 |
| `罐存` / `库容率` | **日度** | 具体日期或近 N 日 |
| `资源池价格` | **季度** | "季度" 表达 |

❌ 严禁犯的错:
- 用户说"Q3 市场份额",清洗成"2024 年 7 月市场份额"(季度被改成月度)
- 用户说"近一周罐存",清洗成"近一个月罐存"(日度被改成月度)

### 2.3 气源国规范化

`澳大利亚` / `卡塔尔` / `马来西亚` / `印尼` / `尼日利亚` / `俄罗斯` / `秘鲁` / `阿曼` / `塞内加尔` / `其他`

| 用户别名 | 规范化 |
|---------|-------|
| 澳洲 / Australia | 澳大利亚 |
| 卡塔 / Qatar | 卡塔尔 |
| 马来 / Malaysia | 马来西亚 |
| 印度尼西亚 / Indonesia | 印尼 |

### 2.4 进口类型识别

- `长协` / `长期协议` / `长期合同` → "长协"
- `现货` / `即期` / `Spot` → "现货"
- 同时问长协 + 现货 → "长协与现货"(保留双指标)

### 2.5 资源池区域规范化

`华南` / `华东` / `华北` / `福建`——只有这 4 个区域。用户说其他区域(如"华中") → 中止清洗,提示"资源池价格仅覆盖华南/华东/华北/福建"。

### 2.6 时间范围补全

| 用户表达 | 清洗后 |
|---------|-------|
| 近 3 月 | 过去三个月 |
| 今年以来 | 2026 年 1 月至今 |
| 去年 | 2025 年全年 |
| 未提到时间 | (不补) |

### 2.7 清洗示例

| 原始用户问句 | 清洗后 query |
|-------------|------------|
| 中海油 2024 LNG 进口 | 2024 年中海油 LNG 进口量(月度,分气源国,长协与现货) |
| 气源结构占比 | 2024 年中海油气源结构市场份额占比(季度) |
| 澳洲 LNG 占比 | 2024 年澳大利亚气源 LNG 进口占比 |
| 近 3 月毛利 | 过去三个月中海油气电集团月度营收与毛利 |
| 华南资源池价 | 2024 年华南区域资源池价格(季度) |
| 大鹏罐存 | 过去一周大鹏接收站罐存量与库容率 |
| Q3 市场份额 | 2024 年第三季度中海油气源市场份额 |

## 三、Step 2 智能体发现规范

### 3.1 keyword 选择策略

本 skill 固定用 `"中海油"` 作为搜索关键词。

| Query 主题 | keyword | 备注 |
|-----------|--------|------|
| LNG 进口 / 销售 / 罐存 / 营收 / 市场份额 / 资源池 / 管道气价 | `"中海油"` | 命中"中海油经营智能体"、"中海油气电分析智能体"等 |

**🔧 后续修改 keyword 的入口**:
- 本文 § 3.2 调用模板
- `skills/public/market-intelligence-company/SKILL.md § Step 2`

如未来 MCP 网关里智能体命名变化(如改成"气电集团经营智能体"),把这两处的 `"中海油"` 改成新词即可。也可以根据查询类型(进口 / 销售 / 罐存)细化为多个 keyword 分发,但当前一个智能体覆盖全部经营场景。

### 3.2 调用模板

```json
{
  "agentListRequest": {
    "keyword": "中海油",
    "status": "published"
  }
}
```

### 3.3 匹配 / 抽取规则

- 返回数组非空 → 取 `description` / `category` / `tags` 与 query 域最贴合的一项(经营 / 进口 / 销售等);并列时取最新 `updateTime`
- 抽取的 **`id` 字段(字符串形态)即后续 `agentId`**
- 返回 `[]` → 回退 `keyword="经营"` 再搜;仍为空则 `keyword=""`;最终为空中止,输出"暂无可用智能体,数据查询不可用"

### 3.4 禁止动作

- ❌ LLM 凭空写 `agentId="1"` / `"2"` 等固定值(经营审计要求每次都用实际发现的 agentId)
- ❌ 同一问题多次调 `listAgentsToolCallback`(发现 1 次后 turn 内复用)
- ❌ 用 `status="draft"` 拉草稿智能体(经营生产场景禁用草稿数据源)

## 四、Step 3 NL2SQL 调用规范

### 4.1 调用模板

```json
{
  "nl2SqlRequest": {
    "agentId": "<Step 2 抽出的 id>",
    "naturalQuery": "<Step 1 清洗后的 query>"
  }
}
```

### 4.2 返回处理

- 返回纯 SQL 字符串 → 作为 Step 4 入参,**不改写**
- 返回空字符串 / 明显错误 → 中止 Step 4,输出"SQL 生成异常,请缩窄 query 范围"

### 4.3 安全约束(经营场景加强)

- **🚨 SQL 字符串永不向用户展示**(经营数据表结构含敏感字段,如 `cnooc_import.spot_premium`、`cnooc_profit.gross_margin_pct` 等)
- **🚨 LLM 不充当 SQL 优化器**,原样透传
- **🚨 不要把 SQL 截图 / 拼接进 Markdown 代码块**

## 五、Step 4 SQL 执行规范

### 5.1 调用模板

```json
{
  "executeSqlRequest": {
    "agentId": "<同 Step 3>",
    "sql": "<Step 3 返回的 SQL 字符串>"
  }
}
```

### 5.2 返回处理

- 返回 JSON 行数组(每行字段名由 SQL 列名决定)→ 进入 Step 5
- 行数 ≥ 3 → markdown 表格呈现
- 行数 ≤ 2 → 自然语言句呈现
- 行数 = 0 → "未检索到对应数据,请调整时间 / 气源国 / 业务类型范围",**禁编造行**(经营审计红线)
- 执行报错 → "数据执行异常",**禁回退到上次结果**

### 5.3 禁自行聚合(经营场景关键)

SQL 已经决定是否聚合(SUM / AVG / GROUP BY 由 NL2SQL 智能体生成):

- ✅ 如果返回行已经是聚合结果 → 直接呈现
- ✅ 如果返回行是明细行 → 直接呈现明细,不补汇总
- ❌ **严禁**自行对返回行做 SUM / AVG / 排名(LLM 算术常幻觉,经营场景下尤为危险——如把真实合计 192 / 119 / 142 错算成 285 / 268 / 276,直接造成审计风险)

## 六、判断算法

```
1. 接收用户原句 → §2 清洗(特别注意 数量 vs 占比 / 时间粒度 / 气源国规范化)
2. listAgentsToolCallback(keyword="中海油", status="published")
   ├─ 非空 → 取最契合的 id → 缓存为 agentId
   └─ 空 → 回退 keyword="经营" → keyword="" 再搜
           ├─ 非空 → 取首项 id
           └─ 空 → 中止
3. nl2SqlToolCallback(agentId, naturalQuery) → 拿到 SQL
4. executeSqlToolCallback(agentId, sql) → 拿到行数据 → Step 5 数据明细
5. 若用户问题需研判 → Step 6 MRTF(web_search 调 1 次)
```
