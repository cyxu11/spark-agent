# 国内域 NL2SQL 链路调用指南

> 国内域数据查询底层走 **3 工具串行链路**:agent 发现 → NL2SQL 生成 → SQL 执行。本文聚焦每个环节的入参 / 出参 / 异常处理规范,**特别关注 `region` vs `sales_unit` 在 query 清洗阶段的识别**。

## 一、工具链概览

| 顺序 | 工具名 | 入参 | 返回 | 单次调用 |
|------|-------|-----|------|---------|
| Step 2 | `listAgentsToolCallback` | `{agentListRequest: {keyword, status}}` | 智能体数组(含 `id` / `name` / `description` / `category` / `tags`) | 每问题 1 次 |
| Step 3 | `nl2SqlToolCallback` | `{nl2SqlRequest: {agentId, naturalQuery}}` | SQL 字符串 | 每问题 1 次 |
| Step 4 | `executeSqlToolCallback` | `{request: {agentId, sql}}` | 行数据(JSON 数组) | 每问题 1 次 |

辅助工具:`web_search`(本平台内置联网搜索)仅在 Step 6.1 MRTF 分析阶段调用 1 次,用于 M / R 维度。

## 二、Step 1 query 清洗规则

### 2.1 🔴 `地理区域` vs `销售/运营单位`(国内域专属)

| 用户提到 | 应识别为 | 清洗示例 |
|---------|---------|---------|
| `华东` / `华南` / `华北` / `山东` / `河北` / `江苏` / `上海` / `北京` / `广东` / `浙江` 等 | **地理区域** | `"华东挂牌价"` → 保留"华东"地理筛选 |
| `中海油` / `中石油` / `中石化` / `昆仑能源` / `申能` / `新奥` 等集团/公司 | **销售/运营单位** | `"中海油接收站挂牌价"` → `"中海油销售单位 LNG 挂牌价"` |
| "接收站" / "销售单位" / "集团" / "公司"类指代 | **销售/运营单位** | `"申能 LNG"` → `"申能销售单位 LNG 价格"` |

❌ 严禁犯的错:
- 用户说"中海油接收站",清洗成"全国挂牌价"或"中海油地区挂牌价"(中海油不是地理名)
- 用户说"申能",识别成省份

### 2.2 时间范围补全

| 用户表达 | 清洗后 |
|---------|-------|
| 近一周 / 上周 | 过去一周 |
| 近一月 / 近 3 月 / 过去一个月 | 过去一个月 / 过去三个月 |
| 今年以来 | 2026 年 1 月至今 |
| 去年 | 2025 年全年 |
| 未提到时间 | (不补) |

### 2.3 气态类型识别(消费量场景)

`气态` / `液态` / `管道气` / `国产气` 在 query 中保留原词,作为消费量分组维度。

### 2.4 消费领域识别(消费结构场景)

`工业` / `发电` / `居民` / `化工` 是消费结构的 4 大领域,保留原词。

### 2.5 清洗示例

| 原始用户问句 | 清洗后 query |
|-------------|------------|
| 近一周国内 LNG 挂牌价 | 过去一周国内 LNG 挂牌价 |
| 中海油接收站近期挂牌价 | 过去一周中海油销售单位 LNG 挂牌价 |
| 华东华南价差 | 过去一周华东与华南 LNG 挂牌价价差 |
| 2024 工业用气占比 | 2024 年工业用气消费占比 |
| 江苏消费排名 | 2024 年江苏天然气消费量及全国排名 |
| 近3月华东液态 LNG 消费 | 过去三个月华东液态 LNG 消费量 |

## 三、Step 2 智能体发现规范

### 3.1 keyword 选择策略

本 skill 固定用 `"国内"` 作为搜索关键词。

| Query 主题 | keyword | 备注 |
|-----------|--------|------|
| LNG 挂牌价 / 区域挂牌价 / 集团挂牌价 | `"国内"` | 命中"国内 LNG 智能体"、"国内市场智能体"等 |
| 全国 / 月度 / 区域消费 / 消费结构 | `"国内"` | 同上 |

**🔧 后续修改 keyword 的入口**:
- 本文 § 3.2 调用模板
- `skills/public/market-intelligence-domestic/SKILL.md § Step 2`

如未来 MCP 网关里智能体命名变化(如改成"中国天然气智能体"),把这两处的 `"国内"` 改成新词即可。

### 3.2 调用模板

```json
{
  "agentListRequest": {
    "keyword": "国内",
    "status": "published"
  }
}
```

### 3.3 匹配 / 抽取规则

- 返回数组非空 → 取 `description` / `category` / `tags` 与 query 域最贴合的那个;并列时取最新 `updateTime`
- 抽取的 **`id` 字段(字符串形态)即后续 `agentId`**
- 返回 `[]` → 退化为 `keyword=""` 再搜一次;仍为空则中止,输出"暂无可用智能体,数据查询不可用"

### 3.4 禁止动作

- ❌ LLM 凭空写 `agentId="1"` / `"2"` 等固定值
- ❌ 同一问题多次调 `listAgentsToolCallback`(发现 1 次后 turn 内复用)
- ❌ 用 `status="draft"` 拉草稿智能体

## 四、Step 3 NL2SQL 调用规范

### 4.1 调用模板

```json
{
  "nl2SqlRequest": {
    "agentId": "<Step 2 抽出的 id>",
    "naturalQuery": "<Step 1 清洗后的 query>"
  }
}
```

### 4.2 返回处理

- 返回纯 SQL 字符串 → 作为 Step 4 入参,**不改写**
- 返回空字符串 / 明显错误(`SELECT *` 无 WHERE / 引用不存在表名)→ 中止 Step 4,输出"SQL 生成异常"

### 4.3 安全约束

- **🚨 SQL 字符串永不向用户展示**
- **🚨 LLM 不充当 SQL 优化器**,原样透传

## 五、Step 4 SQL 执行规范

### 5.1 调用模板

```json
{
  "request": {
    "agentId": "<同 Step 3>",
    "sql": "<Step 3 返回的 SQL 字符串>"
  }
}
```

### 5.2 返回处理

- 返回 JSON 行数组(每行字段名由 SQL 列名决定)→ 进入 Step 5
- 行数 ≥ 3 → markdown 表格呈现
- 行数 ≤ 2 → 自然语言句呈现
- 行数 = 0 → "未检索到对应数据,请调整时间/区域/销售单位范围",**禁编造行**
- 执行报错 → "数据执行异常",**禁回退到上次结果**

### 5.3 禁自行聚合

SQL 已经决定是否聚合(SUM / AVG / GROUP BY 由 NL2SQL 智能体生成):

- ✅ 如果返回行已经是聚合结果 → 直接呈现
- ✅ 如果返回行是明细行 → 直接呈现,不补汇总
- ❌ **严禁**自行对返回行做 SUM / AVG / 排名

## 六、判断算法

```
1. 接收用户原句 → §2 清洗(特别注意 region vs sales_unit 判定)
2. listAgentsToolCallback(keyword="国内", status="published")
   ├─ 非空 → 取最契合的 id → 缓存为 agentId
   └─ 空 → 回退 keyword="" 再搜
           ├─ 非空 → 取首项 id
           └─ 空 → 中止
3. nl2SqlToolCallback(agentId, naturalQuery) → 拿到 SQL
4. executeSqlToolCallback(agentId, sql) → 拿到行数据 → Step 5 数据明细
5. 若用户问题需研判 → Step 6 MRTF(web_search 调 1 次)
```
