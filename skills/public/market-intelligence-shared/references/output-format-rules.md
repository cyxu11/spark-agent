# 市场态势-输出格式统一规范(单一权威源)

> ⚠️ 本文件是全部市场态势业务 skill 共享的输出禁令权威源。各 skill 在自己的 SKILL.md 中只点最相关红线,详细禁令清单与反向示例统一在本文件维护。

## 角色背景

本文件约束 `market-intelligence-*` 系列业务 skill 的输出格式。Agent 输出**直接展示给终端业务用户**(交易员、市场分析师、战略规划人员、央企合规审计),前端为对话气泡,marked + DOMPurify 渲染,无任何后处理 middleware。

---

## 一、7 道输出禁令

### 禁令 1:禁伪表格 / HTML 表格(分析段同时禁 markdown 表格;数据段豁免 markdown 表格)

> ⚠️ **重要变更(R2.12-patch1)**:**数据检索段(retriever 风格)**豁免「禁 markdown 表格」,改为**优先用 GFM 标准 markdown 表格**展示结构化数据。**MRTF 分析段(analyst 风格)**仍严格禁表格——T/F 章节按 R2.13 豁免允许有限 markdown 表格。

❌ **全场景禁止**:HTML `<table>`/`<tr>`/`<td>`/`<th>` 标签
❌ **全场景禁止**:视觉伪表格,如 `日期: 2026-04-23 | 市场: JKM | 价格: 16.39`(单行 `|` 拼接)
❌ **全场景禁止**:换行枚举伪表格,如:
```
2026-04-23: 16.39
2026-04-22: 16.13
```

#### 1-A. MRTF 分析段(analyst 风格)— 严格禁 markdown 表格(R2.13 T/F 章节有限豁免)

❌ 默认禁止任何 `|` 列分隔符或 `---|---` 分隔线
✅ 改为自然语言:`近 5 日 JKM 价格依次为 16.39、16.13、15.81、15.24、15.00 美元/MMBtu(按日期倒序),累计下行 8.5%`
✅ R2.13 豁免:**T 技术面 / F 基本面章节**允许 GFM 标准 markdown 表格,数据行 ≥ 3(双边对比 ≥ 2)

#### 1-B. 数据检索段(retriever 风格)— ✅ 豁免:可用 markdown 表格(R2.12-patch1)

✅ **优先**用 GFM 标准 markdown 表格(`|` 列分隔 + `---|---` 分隔线)展示 ≥ 3 行结构化数据
✅ 表头第 1 行用业务列名(中文),第 2 行 `|---|---|`(必填,GFM 标准)
✅ ≤ 2 行单条数据可继续用自然语言段落
✅ 仍**禁伪表格**

✅ 正确示例(数据段输出):
```
| 日期 | 市场 | 价格(美元/MMBtu) | 涨跌幅 |
|------|------|--------|--------|
| 2026-04-23 | JKM | 16.39 | -1.59% |
| 2026-04-22 | JKM | 16.13 | -1.95% |
| 2026-04-21 | JKM | 15.81 | +3.74% |

数据来自国际天然气价格数据,共 120 条,呈现最近 3 条。
```

### 禁令 2:禁 `#` 标题语法(MRTF 5 H3 块豁免)

❌ 默认禁止 `#`/`##`/`####`/`#####`/`######`
✅ 用 `**加粗小节名**:` 替代,如 `**趋势研判**:近 5 日 JKM 价格…`
✅ 或全段叙述,不分小节

**R2.13 MRTF 豁免**:MRTF 分析段允许 5 个 H3 章节标题:
- `### 一、宏观面 (M-Macro)` / `### 二、政策面 (R-Regulation)` / `### 三、技术面 (T-Trend)` / `### 四、基本面 (F-Fundamental)`
- 末段「综合研判」用 `**综合研判**:` 加粗节名,**不**用 `### 五、综合研判` H3

❌ 禁阿拉伯数字编号(`### 1. 宏观面`)/ 禁罗马数字(`### Ⅰ、宏观面`)/ 禁纯英文(`### M. Macro`)— **S1 约束**(仅中文数字 一/二/三/四 合规)

### 禁令 3:禁代码块

❌ 禁止 ` ``` ` 三反引号代码块包裹任何内容(SQL、JSON、配置)
✅ 如需引用单个标识符,用单反引号 `code` 内联

### 禁令 4:禁暴露内部信息(脱敏)

❌ 禁止输出 SQL 语句
❌ 禁止暴露数据源表名(如 `builtin_natural_gas_prices`、`builtin_lng_listing_prices`、`builtin_cnooc_*`)
❌ 禁止暴露字段英文名
❌ 禁止暴露 MCP 工具名(如 `mcp_market_gas_prices`、`mcp_iflytek_cbm`、`mcp_price_converter`)
❌ 禁止暴露内部组件名(`input_parser` / `team1` / `team2` / `team3` / `team4` / `sub-1-1` / `Step 0` / `_inject_retriever_outputs` 等)
✅ 本地数据脱敏为「据本平台数据」「据国际天然气价格数据」「据国内 LNG 挂牌价数据」「据本平台换算工具」
✅ 联网结果脱敏为「据[机构/媒体名称]」(如「据路透社」「据 IEA」「据国家能源局」「据发改委」)

### 禁令 5:禁引用图表/附件

❌ 禁止「趋势可见下方图表」「如下图所示」「详见附图」「图表见后」「示意图见下」等引用表述
**原因**:Agent 输出是纯文字,前端不渲染任何图表/附件,此类引用会让用户困惑
✅ 直接用文字描述趋势

### 禁令 6:禁过程语 / 引导语 / 追问建议

❌ 禁开头写「让我查询…」「正在分析…」「根据您的问题…」等过程语
❌ 禁结尾写「追问建议」「您可能还想了解」等引导语
❌ 禁输出 JSON 代码、占位符(如 `[价格]`、`[日期]`)
✅ **首句即结论**,直接给答案或核心判断
✅ 直接结束分析,不附加引导

### 禁令 7:MRTF 分析段禁前缀段 / 前缀句 / 前缀引导语

❌ 禁止在首段加粗小节名(`**市场研判**:` / `**经营研判**:` / `**换算研判**:` / `**综合研判**:`)之前输出任何前缀段 / 前缀句 / 前缀引导语 / 前缀汇总句(如「据网络搜索结果…」「综合上述信息…」「以下为分析…」「先看…」「概览如下…」)
❌ **`**xx研判**` 前一字符必须为 `\n` 或文档第 0 字符**(R2.9 patch3 主约束 — 兜住「共 120 条记录,仅呈现最近 5 条 **市场研判**:…」截图复现场景);空格、逗号、句号、冒号、任何非换行字符都不允许

✅ 正确示例 1(直接起首):`**市场研判**:近期 JKM 价格震荡下行…`
✅ 正确示例 2(前面有 `\n`):`共 120 条记录,仅呈现最近 5 条\n\n**市场研判**:近期 JKM 价格…`

**适用范围**:所有 MRTF 分析段(国际市场分析 / 国内市场分析 / 经营数据分析);数据检索段输出不适用本禁令。

## 二、Markdown 允许子集(仅以下三项 + R2.13 豁免)

| 允许项 | 用途 | 示例 |
|--------|------|------|
| 加粗 `**关键数字**` | 突出价格、百分比、关键结论 | `**当前 JKM 价格 16.39 美元/MMBtu**` |
| 段落 + 空行分隔 | 让 marked 正确渲染段落 | 连续文字 + 空行换段 |
| 内联引用 | 自然语言外部引用 | `据路透社` / `据本平台数据` |

**R2.13 MRTF 豁免**:
- ✅ 5 个 H3 章节标题(中文数字编号)
- ✅ T/F 章节 GFM 标准 markdown 表格(数据行 ≥ 3,双边对比 ≥ 2)

## 三、数据来源标识硬约束(R2.14-Patch1)

### `[数据]` 标识 — 本平台权威数据来源

**MRTF 分析段**:每次引用本平台数据明细的具体数值(价格 / 同比 / 库存 / 价差等),**必须在该数值或该句陈述末尾追加 `[数据]` 标识,不得省略**。

✅ 正面示例:`据本平台数据,JKM 现货 16.39 美元/MMBtu [数据],近 7 日下行 3.2% [数据]`
✅ 正面示例:`据本平台数据,Brent 当前 82.15 美元/桶 [数据],较上月 +1.8% [数据]`
❌ 反面示例:`据本平台数据,JKM 现货 16.39 美元/MMBtu,近 7 日下行 3.2%`(缺 `[数据]`,违反硬约束 → 整段重写)

**适用范围**:T 技术面 + F 基本面章节中所有引用本平台数据的具体数值;Markdown 表格自身的数据行不重复加 `[数据]`,但表格下方综述句仍需追加。M / R 章节联网搜索得到的数值用 markdown link `据 [机构](URL)`,不需 `[数据]` 标识。

### `据 [机构](URL)` — 联网搜索来源

**关键约束**:仅当联网搜索工具实际返回了某机构/媒体的内容时,才能在正文用 markdown link 格式 `据 [机构名](URL) 报道`(URL 来自 candidate_pool);**没有真实工具返回时,只写「据本平台数据」/「据国际天然气价格数据」**,禁假写「据路透社/彭博社/IEA」等。

❌ 禁裸 URL / 禁空 link `[机构]()` / 禁假写机构名

## 四、量化输出风格规范

| 维度 | 规则 |
|------|------|
| 首句 | 直接给结论或核心数值,禁过程语 |
| 长度 | 数据段整体 2-5 行;分析段(MRTF)整篇 800-1500 字 ±20% |
| 综合研判段 | 150-300 字,首句即结论(R2.13 升级,原 100-250 字) |
| 价格精度 | 保留 2 位小数(`16.39 美元/MMBtu`)/ 国内挂牌价整数(`5800 元/吨`) |
| 百分比精度 | 保留 1 位小数 + `%`(`下行 8.5%`) |
| 汇率精度 | 保留 4 位小数(`7.2500`) |
| 时间表达 | 用「近 N 日 / 本月 / 同比上月 / 保供季」等业务表达 |
| 数据来源 | 本地→「据本平台数据」/「据 XX 数据」;联网→「据[机构/媒体]」 |

## 五、反向示例集

### 场景 A-1:MRTF 分析段输出 markdown 表格(违反禁令 1-A 默认,需 T/F 章节才豁免)

❌ 错误(M/R 章节用表格):
```
| 日期 | 市场 | 价格 |
|------|------|------|
| 2026-04-23 | JKM | 16.39 |
```

✅ 正确(M/R 用自然语言):
**市场研判**:近 5 个交易日 JKM 价格依次为 16.39、16.13、15.81、15.24、15.00 美元/MMBtu(按日期倒序),累计下行 8.5%。

### 场景 A-2:数据段输出 markdown 表格(✅ R2.12-patch1 推荐)

✅ 正确(retriever 段,≥ 3 行数据优先表格):见禁令 1-B 示例。

### 场景 B:伪表格(竖线分隔)

❌ 错误:
```
日期: 2026-04-23 | 市场: Brent | 价格: 82.15
日期: 2026-04-22 | 市场: Brent | 价格: 81.50
```

✅ 正确:近 5 日 Brent 价格在 80.30-82.15 美元/桶区间震荡,最新报 82.15(2026-04-23),周环比上行 1.8%。

### 场景 C:HTML 表格

❌ 错误:`<table><tr><th>市场</th><th>价格</th></tr><tr><td>JKM</td><td>16.39</td></tr></table>`

✅ 正确:当前 JKM 报 16.39 美元/MMBtu,TTF 报 44.50 欧元/MWh(约 47.80 美元/MMBtu),亚洲溢价约 5%。

### 场景 D:暴露表名/工具名

❌ 错误:`查询 builtin_natural_gas_prices 表,调用 mcp_iflytek_cbm 得到…`

✅ 正确:据国际天然气价格数据,JKM 当前报 16.39 美元/MMBtu;据路透社最新报道,亚洲补库存推动价格走高。

### 场景 E:`#` 标题紧贴段落

❌ 错误:`当前价格为 12.5 美元/MMBtu #### 趋势研判`

✅ 正确:当前 JKM 价格 12.50 美元/MMBtu。**趋势研判**:近 5 日呈下行态势,累计跌幅 2.3%。

### 场景 F:研判段前缀(违反禁令 7)

❌ 错误 1:`据网络搜索结果,近期 JKM 价格震荡下行,可能是主要原因。**市场研判**:…`(前缀段冗余)
❌ 错误 2:`综合上述信息,JKM 走势如下:**市场研判**:…`(前缀句冗余)
❌ 错误 3:`共 120 条记录,仅呈现最近 5 条 **市场研判**:近期 JKM…`(数据陈述句末尾紧跟空格 + `**市场研判**:`,前一字符是空格不是 `\n`)

✅ 正确:`共 120 条记录,仅呈现最近 5 条\n\n**市场研判**:近期 JKM…`(数据陈述独立成段)
✅ 正确:`**市场研判**:近期 JKM 价格震荡下行…`(直接以研判加粗段起首,第 0 字符 = `*`)

## 六、数据检索段特例(R2.12-patch1)

数据检索段(`data retriever` 行为)输出**直接展示给真人用户**,约束:
- **禁令 1-B**(豁免 markdown 表格):≥ 3 行结构化数据优先 GFM 标准 markdown 表格;≤ 2 行单点数据用自然语言
- **禁令 2-7 全部生效**(禁 `#` 标题 / 代码块 / 暴露内部信息 / 引用图表 / 过程语;禁令 7 不适用)
- **数据来源脱敏**:本地→「据 XX 数据」/ 联网→「据[机构/媒体]」
- 数值保留工具返回原始精度,禁四舍五入
- 数据点超过 5 个时仅传最近 5 条 + 标注「共 N 条记录」
- **禁输出趋势性分析结论**:不得输出"累计下行/累计上行/同比上行"等趋势性判断 — 仅呈现原始数据 + 数据范围说明,趋势分析放在 MRTF 分析段
