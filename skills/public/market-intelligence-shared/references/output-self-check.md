# 输出前自检清单(LLM 自规范化)

> 项目没有 post-processing middleware。原产品交付的 13 条 post_hook 正则规则,转化为 LLM 在输出**前**的自检项。生成最终答案前,逐项核对并自修正,确保 marked + DOMPurify 渲染正常。

## 一、Markdown 语法规范

| 检查项 | 错误形态 | 正确形态 |
|--------|---------|---------|
| 标题与文字之间空格 | `####标题` | `#### 标题`(`#` 后必有空格) |
| 标题前空行 | `文字\n####` | `文字\n\n####` |
| 标题与前文不同行 | `句号。#### 趋势` | `句号。\n\n#### 趋势` |
| 列表符号统一 | `– 条目` / `— 条目`(em-dash) | `- 条目` |
| 中文有序列表 | `1、条目` | `1. 条目` |
| 中文一二三列表 | `一、条目` / `（一）条目` | `- 条目` |
| 加粗节名前空行 | `文字\n**综合研判**:` | `文字\n\n**综合研判**:` |
| 行尾空格 | `文字   ` | `文字`(无尾随空格) |

## 二、表格规范(retriever 段 + MRTF 的 T/F 章节)

| 检查项 | 处理方式 |
|--------|---------|
| 占位符行 `\| --- \|` 或 `\| ——— \|` 全空 | 删除该行(保留合法 `\|---\|---\|` 分隔行) |
| 表格行间空行 | 收拢为单换行,避免破坏 GFM 渲染 |
| HTML `<table>` 标签 | 改写为 markdown 表格语法 |
| 数据行 ≥ 3(不含表头) | < 3 行 → 改为自然语言段落或合并对比表(双边场景 ≥ 2) |

## 三、研判段独立约束

**`**综合研判**` / `**市场研判**` / `**经营研判**` / `**换算研判**` 前一字符必须为 `\n` 或文档第 0 字符**

❌ 错误:`共 120 条记录,仅呈现最近 5 条 **市场研判**:近期…`(前一字符是空格)
✅ 正确:`共 120 条记录,仅呈现最近 5 条\n\n**市场研判**:近期…`(前一字符是 `\n`)

紧贴非换行字符时,在前面强制补 `\n\n` 独立成段。

## 四、`[数据]` / `[来源]` 标识规范化

| 错误形态 | 正确形态 |
|---------|---------|
| `【数据】` / `(数据)` / `（数据）` | `[数据]`(本平台数据标识) |
| `【来源】` / `(来源)` / `（来源）` | `[来源]`(联网搜索来源标识) |

`[数据]` 标识只能跟在数值或本平台数据陈述句末尾;不滥用于无具体数值的方向性陈述。

## 五、内部信息脱敏检查(禁令 4)

| 必须脱敏的内容 | 替换为 |
|--------------|-------|
| 表名 `builtin_natural_gas_prices` / `builtin_lng_listing_prices` / `builtin_cnooc_*` 等 | 「据本平台数据」/「据国际天然气价格数据」/「据国内 LNG 挂牌价数据」 |
| 字段英文名 `volume_mt` / `avg_price` / `share_pct` 等 | 「进口量」/「均价」/「占比」 |
| MCP 工具名 `mcp_market_gas_prices` / `mcp_iflytek_cbm` / `mcp_price_converter` 等 | 「据本平台数据」/「据联网搜索」/「据本平台换算工具」 |
| 内部组件名 `input_parser` / `team1`-`team4` / `sub-1-1` / `Step 0` / `_inject_retriever_outputs` | 全部删除,不出现在用户可见输出中 |
| SQL 语句 | 全部删除 |

## 六、特殊红线

- ❌ `据网络搜索结果,…` / `综合上述信息,…` / `以下为分析,…` 等前缀段(违反禁令 7)
- ❌ 编造 URL / 空 link `[机构]()` / 假写「据路透社/IEA」(无真实工具返回时)
- ❌ 「相关数据」「具体数据」「详细数据」「相应信息」等空话词代替具体数值

## 七、最终一致性检查

提交前,从用户视角通读一遍:
- 我能不能从这段文字一眼看到核心结论?(否则首句不达标)
- 我能不能数出至少 3 处具体数值?(否则违反 R2.14-Patch1)
- 数值我能不能找到来源?(本平台/机构 markdown link)
- 有没有任何字符串看起来像内部命名(下划线/英文表名)?(否则脱敏不达标)
- 表格是不是有 `|---|---|` 分隔行?(否则 marked 不会渲染表格)

▎ 任意一条不达标 → 修正后再输出。
