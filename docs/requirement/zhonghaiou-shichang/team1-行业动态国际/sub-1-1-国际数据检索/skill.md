## 量化硬性规则

1. **优先 markdown 表格**:≥ 3 行结构化数据必用 GFM 表格(双端 marked 自动渲染为 HTML `<table>`)
2. **表头中文业务列名**:如 `日期`、`市场`、`价格(美元/MMBtu)`、`涨跌幅`,禁用英文字段名
3. **分隔行必填**:`|---|---|` 是 GFM 标准,前端无此行不会渲染为表格
4. **数据精度**:价格保留 2 位小数,百分比带 `%` 保留 1 位小数
5. **时间表达**:用「近 N 日 / 本周 / 同比上月」等业务表达
6. **长度**:表格 ≤ 5 行 + 1 句总结(共 N 条记录,呈现最近 K 条)
7. **数据来源脱敏**:表格之后/之前的总结句写「据国际天然气价格数据」「据本平台数据」,禁写表名
8. **禁输出趋势性分析结论**(R2.12-patch1 增补):不得输出"累计下行/累计上行/累计跌幅/累计涨幅/累计回落/累计回升/同比上行/同比下行"等趋势性分析结论;只输出原始数据(表格/单点)+ 数据范围说明(据 XX 数据,共 N 条记录,呈现最近 K 条)。趋势分析由下游 sub-1-2 完成。
9. **🔴 多维数据强制聚合**(R1.7-DQ-Fix):若工具返回中包含 `aggregated_summary` 字段(对 lng_trade 等多维表生效),**必须**直接读 `aggregated_summary.by_time` 数组的 `total_*` 字段,**严禁**自行对 `result` 多维明细行做 SUM 计算(LLM 算术经常幻觉)。

## 避免(详见 _shared/output-format/rules.md 禁令 1-B)

- ❌ HTML `<table>` 直接写(必须用 markdown 表格)
- ❌ 伪表格(`日期: 值 | 价格: 值` 单行 或 `2026-04-23: 16.39` 多行枚举)— 缺分隔行 `|---|`
- ❌ 暴露 `builtin_natural_gas_prices` 等表名 / 字段英文名 / `mcp_iflytek_cbm` 等 MCP 工具名 / SQL
- ❌ `#` 标题语法 / ` ``` ` 代码块
- ❌ 引用图表/附件(「如下图」「详见附图」「趋势可见下方图表」)
- ❌ 编造数据 / 占位符(`[价格]`、`[日期]`)
- ❌ 输出分析结论(分析由下游 sub-1-2 完成,本 sub-agent 只交付数据)
</format>
