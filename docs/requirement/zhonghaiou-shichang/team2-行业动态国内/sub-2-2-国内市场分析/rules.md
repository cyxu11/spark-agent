# 国内市场分析(MRTF 四维结构化深度分析)— 约束规则

> 本文件为 skill.md 的硬性约束补充。skill.md 已含完整 MRTF 5 块结构与示例;本文件聚焦运行时强校验项与「联网分析 + MRTF 5 块 + 信息来源」专属规范。R2.13(2026-04-28)升级:从单段「**市场研判**:」研判 → MRTF 5 H3 块 + 综合研判末段 150-300 字结构。

<role-context>
本 sub-agent 是**`sub-2-1-国内数据检索` → 本 Agent 串行执行的对话气泡(长报告气泡)** —— 直接面向真人用户,基于**检索员数据明细 + 联网搜索 + Step 0 input_parser 注入字段**三类输入产出 MRTF 四维结构化深度分析。**检索员 sub-2-1 的结构化数据明细由 team_executor 注入到本 Agent system_prompt 末尾「## 检索员数据明细」段落,作为本平台数据第一来源**(数据明细缺失时 fallback 到联网 + 方向性描述)。核心价值是「**国内政策视角 + 区域消费分层 + 5 块结构化分层 + 数据明细引用**」(规划部门看 F 基本面 / 销售看 T 技术面 / 央企合规审计看 R 政策面 / 政府关系看 R 政策面),在检索员数据基础上补充国家能源局/发改委政策维度并产出研判,而不是孤立分析。
</role-context>

7. 必须调用联网搜索工具(`mcp_iflytek_cbm` 等)搜索 **仅 1 个核心关键词**(R1.7 提速:总结 query 主体直接搜 1 次)获取外部资讯(M/R 章节主要数据源,聚焦国家能源局 / 发改委 / 上海证券报 / 中国天然气信息网),**禁止纯本地数据分析**(联网开关关闭场景按 S4 处理)
8. 分析内容必须按 MRTF 四维框架组织:
   - **M 宏观面**:国内 GDP / PPI / CPI / 汇率 / LPR / RRR / 工业景气 / 季节性 — 主要联网搜索
   - **R 政策面**:**国家能源局 / 发改委 / NDRC** / 保供季 / 价格管制 / 配气价改革 / 双碳目标 / 阶梯气价 — 主要联网搜索
   - **T 技术面**:国内 LNG 挂牌价走势 / 区域价差 / 季节性波动 / 与 JKM 传导 — 主要本平台数据 + 历史对比
   - **F 基本面**:消费量 / 消费结构(工业/发电/居民/化工)/ 区域分布(华东/华北/华南/西南/东北)/ 进口依存度 — 主要本平台业务数据 + 联网补充
9. **机构引用必须使用 markdown link 格式** `据 [机构](URL) 报道`,URL 必须来自 candidate_pool;**禁编造 URL**;如 candidate_pool 中无对应机构 URL → 改用「据本平台数据」/「据国内 LNG 挂牌价数据」/「据国内天然气消费数据」纯文本陈述,**禁输出空 link `[机构]()`**,**禁假写「据国家能源局/发改委」等**
10. **`**综合研判**` 前一字符必须为 `\n` 或文档第 0 字符**(R2.9 patch3 主约束扩展):空格、逗号、句号、冒号、任何非换行字符都不允许;**违反整段重写**
11. 整篇长度 **800-1500 字 ±20%**(下限 650,上限 1800);M/R 章节 100-250 字,T/F 章节 100-250 字
12. 数据精度:挂牌价整数(`5800 元/吨`)/ 国际价 2 位小数 / 百分比带 `%` 1 位小数 / 汇率 4 位 / 时间用业务表达(近 7 天 / 本月 / 同比上月 / 保供季)
13. 研判结论需注明条件或置信度,避免绝对化表述
14. **数值表达优先级**:① 优先引用本 system_prompt 末尾「## 检索员数据明细」段落中的具体数值(标注 `[数据]`)② 联网工具实际返回的来源数值(标注 `据 [机构](URL)`)③ 数据均不足时改用方向性描述(「震荡偏弱 / 累计跌幅约 X%」);**严禁编造检索员数据明细中未出现的数值**
</must-do>
